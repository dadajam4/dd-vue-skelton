link Menu and dialog

メニュー、またはダイアログで選択させる事ができます。
この場合 dd-date-picker-fragment で dd-input をラップしてください。
input要素のvalueは自動的にpickerのvalueとして同期されます。
デフォルトではメニュー表示となりますが、ダイアログ表示を行う場合は dialog 属性を設定します。
cancellable 属性を設定する事でキャンセル、OKを選択させる事ができます。
この場合においてはメニューやダイアログエリアの外をクリックした時やESCキー押下においてもキャンセル扱いとなり、メニュー・ダイアログ表示開始時の値にロールバックされます。
persistent 属性とdialogを併用する事で、値の入力を強制させる事が可能です。

link Allowed dates

allowed-dates 属性を利用する事で選択可能な日付を限定できます。
String, Array, Function のいずれかを利用してください。 ex. ['2018-3-1', '2018-1', ...] or info => (info.value === '2012-02-11')

link Holidays

休日表示設定をいくつか行う事が可能です。
holidays
カスタムで休日を設定できます。受け付ける値は String, Array, Function のいずれかです。
holiday-weeks
休日となる固定曜日を設定可能です。
holiday-color
休日を表記する際のカラーを指定可能です。

link Events

イベント日程を設定可能です。
events
イベント日を設定できます。受け付ける値は Array, Object{ value, color }, String のいずれかです。
event-color
イベント日のカラーを設定可能です。デフォルトは warning です。
このカラーはオブジェクトでイベント日を指定する場合、 color キーで上書き可能です。

link Coloring & more

いくつかのカラー設定や、表示オプションがあります。
picked-color
選択した日付のカラーを設定可能です。デフォルトは primary です。
current-color
現在の日付のカラーを設定可能です。デフォルトは primary です。
show-current
現在の日付を強調表示したくない場合、 false を指定します。
show-fill-day
現在の月からはみ出した日付を表示する場合、 true を指定します。

link Multiple

開始日、終了日からなる複数の日付を選択する事が可能です。
この時 v-model は初期値として空の配列である必要がある事に注意してください。
いかなる場合でもインデックス0に開始日、1に終了日が格納されますが、
emptyな状態では空の配列となります。つまり、インデックス0も1も undefined な状態が存在する事を忘れないでください。

multiple で動作をさせる場合、ダイアログ等の補助となる fragment 機構は現在利用できません。
これは入力対象となる v-model を特定する実装を現時点で行っていない事が原因です。
ダイアログやメニューと連動した実装を行うためには dd-menu などを併用する必要があります。
